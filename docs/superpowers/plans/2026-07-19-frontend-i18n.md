# Frontend I18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan phase by phase. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add default-English and persisted Simplified Chinese localization to Flint's frontend.

**Architecture:** Register global `vue-i18n` Composition API resources, persist the locale in the browser, and translate only Flint-owned frontend copy. Stored values, Agent output, technical terms, and server error messages remain unchanged.

**Tech Stack:** Vue 3, TypeScript, Pinia, `vue-i18n`, Bun test, Playwright.

## Global Constraints

- Locales are exactly `en` and `zh-CN`; English is default and fallback.
- The language icon sits beside the theme icon and uses `文` / `A`.
- Preserve the translation boundaries in `docs/superpowers/specs/2026-07-19-frontend-i18n-design.md`.

---

### Phase 1: I18n Foundation

**Files:** Create `apps/web/src/i18n/index.ts`, `apps/web/src/i18n/locales/en.ts`, `apps/web/src/i18n/locales/zh-CN.ts`, and `tests/web/i18n.test.ts`; modify `apps/web/src/main.ts`, `apps/web/src/App.vue`, `apps/web/package.json`, and `bun.lock`.

- [ ] Write failing tests for default/fallback locale, persistence, invalid storage, switching, and `<html lang>`.
- [ ] Install and register `vue-i18n`; implement the typed resources and locale controller.
- [ ] Add the accessible language icon beside the theme icon.
- [ ] Gate: focused i18n tests and web typecheck pass.

### Phase 2: Frontend Copy Migration

**Files:** Modify `apps/web/src/App.vue`; all files under `apps/web/src/views/`; user-facing components under `apps/web/src/components/`; frontend display helpers under `apps/web/src/components/` and `apps/web/src/stores/`; browser notification copy under `apps/web/src/realtime/`; and their existing tests under `tests/web/`.
